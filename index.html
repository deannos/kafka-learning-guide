What is Apache Kafka?

A distributed event streaming platform built for high-throughput, fault-tolerant, real-time data processing at any scale.

🌊
Event Streaming Platform
Kafka enables you to publish, subscribe to, store, and process streams of events in real-time. It's the nervous system of your data infrastructure.
StreamingReal-time
🔗
Distributed by Design
Built to run as a cluster across multiple servers, providing horizontal scalability and fault tolerance through replication.
DistributedFault Tolerant
⚡
Extreme Performance
Sequential disk I/O, zero-copy transfers, and intelligent batching deliver millions of messages per second with sub-millisecond latency.
High ThroughputLow Latency
♾️
Durable Storage
Messages are persisted to disk and replicated. Unlike queues, consumers can replay events days or weeks later with configurable retention.
PersistentReplayable
// THE PROBLEM

Why Kafka Exists

Before event streaming, systems suffered from tight coupling, integration complexity, and scalability ceilings.

⚠ The Integration Problem
With N services, point-to-point integration creates N×(N-1)/2 connections. 50 services = 1,225 pipelines. Each has custom schemas, different failure modes, and independent scaling. This is unsustainable.
❌
Before Kafka
  • Point-to-point integrations between every service pair
  • Data consumers must poll source systems repeatedly
  • No message history — if a consumer is down, data is lost
  • Tight coupling makes changes catastrophically risky
  • Different teams maintain their own integration pipelines
✅
After Kafka
  • Producers write to topics; any consumer can subscribe independently
  • Consumers read at their own pace from persistent log
  • Messages retained for days/weeks — replay anytime
  • Producers don't know about consumers — fully decoupled
  • One central platform serves all teams and use cases
// CORE CONCEPT

Event Streaming

An event is any notification that something happened: a user clicked, an order was placed, a sensor reported temperature. Event streaming captures these notifications in real-time and makes them available to any system that needs them.

📡
Capture
Capture events in real-time from databases, sensors, mobile devices, cloud services, and software applications using producers.
🗄️
Store
Store event streams durably and reliably on a distributed log for as long as you need — hours, days, or forever with log compaction.
⚙️
Process
Process and react to event streams in real-time or retrospectively using Kafka Streams, ksqlDB, or any consumer application.
🚀
Deliver
Route processed events to the right destinations: databases, search indices, analytics systems, or other applications.

Get Running Fast

docker-compose.yml — Kafka + Zookeeper 
version: '3.8'
services:
  zookeeper:
    image: confluentinc/cp-zookeeper:7.5.0
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000

  kafka:
    image: confluentinc/cp-kafka:7.5.0
    depends_on: [zookeeper]
    ports: ["9092:9092"]
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_AUTO_CREATE_TOPICS_ENABLE: 'true'
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
bash — Start, create topic, produce & consume 
# Start Kafka
docker-compose up -d

# Create a topic
kafka-topics.sh --create \
  --bootstrap-server localhost:9092 \
  --topic my-events \
  --partitions 3 \
  --replication-factor 1

# Produce messages (terminal 1)
kafka-console-producer.sh \
  --bootstrap-server localhost:9092 \
  --topic my-events

# Consume messages (terminal 2)
kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic my-events \
  --from-beginning
