At work, our architecture is event-driven. Kafka sits at the core of almost everything we build.

When new engineers joined the team, I kept noticing the same pattern — great developers spending days piecing together knowledge from scattered docs, videos, and blog posts just to get productive with Kafka.

So I built an internal learning resource. Then I thought — why keep it internal?

I've open-sourced it as a free Kafka Developer Learning Portal.

It covers everything our team actually needed to know:

→ Architecture — brokers, topics, partitions, ISR, KRaft vs ZooKeeper
→ Advanced Concepts — exactly-once semantics, Kafka Streams, Schema Registry, Kafka Connect
→ Security — TLS, SASL/SCRAM, ACLs with full production config examples
→ Real-World Use Cases — banking, IoT, e-commerce, CDC, trading systems
→ Performance Tuning — producer & consumer config profiles for different scenarios
→ Operations Runbook — monitoring metrics, failure diagnosis, rolling restarts
→ Kafka vs Others — RabbitMQ, Redis, REST, Pulsar honestly compared
→ CLI Cheatsheet — every command and config in one place
→ Interview Prep — 13 Q&As + interactive quiz
→ Learning Roadmap — 10 milestones, beginner to architect

No frameworks. No login. No paywall. Pure HTML/CSS/JS on GitHub Pages.

If your team is working on event-driven systems and you're onboarding engineers onto Kafka — I hope this saves someone the hours it took us to figure things out.

Feedback from the community is very welcome.
