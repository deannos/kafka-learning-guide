Developer Roadmap

A structured 10-milestone path from Kafka beginner to production architect. Each milestone includes what to learn, what to build, and how to validate your skills.

// PHASE 1 — WEEKS 1-2

Beginner: Foundation

MILESTONE 01
Kafka Fundamentals
Understand what Kafka is, what problems it solves, and when to use it. Learn the core vocabulary: topics, partitions, offsets, producers, consumers, consumer groups, brokers, and clusters. Understand event streaming vs traditional messaging.

Study: Read the official Kafka documentation intro. Watch Confluent's "Kafka in 5 minutes". Understand the commit log concept.

✅ Milestone achieved when: You can explain Kafka to a colleague without looking at notes, and articulate why a system would use Kafka vs RabbitMQ vs REST.
Event Streaming Core Concepts Architecture Overview
MILESTONE 02
Local Setup & First Messages
Get Kafka running locally using Docker Compose. Use the CLI tools to create topics, produce messages, and consume them. Understand the command-line tools: kafka-topics.sh, kafka-console-producer.sh, kafka-console-consumer.sh.

Project: Run Kafka locally, create a "hello-world" topic with 3 partitions, produce 100 messages with keys, consume them and observe partition assignment.
Docker CLI Tools First Messages
MILESTONE 03
Producer & Consumer APIs
Write your first producer and consumer in Python or Java. Understand serialization/deserialization, acks, offset management, auto-commit vs manual commit, and consumer group behavior.

Project: Build a simple e-commerce order pipeline. Producer sends order events, consumer processes them and prints order confirmations. Add error handling and manual offset commits.
Python/Java SDK Serialization Offset Management
// PHASE 2 — WEEKS 3-5

Intermediate: Building Systems

MILESTONE 04
Docker & Production-Like Setup
Set up a multi-broker Kafka cluster using Docker Compose (3 brokers + ZooKeeper or KRaft). Configure replication factors, understand ISR, test broker failure and recovery. Set up Schema Registry and Kafka UI.

Project: Kill a broker and observe that your producer/consumer keep working. Watch leader election happen. Verify data wasn't lost.
Multi-broker Replication Fault Tolerance
MILESTONE 05
Schema Registry & Avro
Integrate Confluent Schema Registry. Define Avro schemas for your events. Understand schema evolution and compatibility modes. Replace JSON with Avro serialization and observe the size reduction.

Project: Migrate your order pipeline to use Avro schemas. Add a new optional field and verify backward compatibility. Try a breaking change and see the registry reject it.
Avro Schema Evolution Data Contracts
MILESTONE 06
Kafka Streams
Build stateful stream processing applications using the Kafka Streams DSL. Learn KStream, KTable, joins, aggregations, and windowing. Understand state stores backed by RocksDB.

Project: Real-time order analytics — count orders per customer in 5-minute windows, join with a customer profile KTable, emit enriched analytics events. Test late-arriving data handling.
KStream/KTable Windowing State Stores Joins
MILESTONE 07
Kafka Connect
Deploy Kafka Connect and configure source and sink connectors. Set up Debezium CDC from a PostgreSQL database. Route changes to Elasticsearch using the Elasticsearch Sink connector.

Project: Build a CDC pipeline: PostgreSQL → Debezium → Kafka → Elasticsearch. Insert/update/delete rows in Postgres and watch search results update in real-time.
Debezium CDC Connectors SMTs
// PHASE 3 — WEEKS 6-10

Advanced: Production Systems

MILESTONE 08
Build a Real System
Build a complete event-driven application with multiple services communicating through Kafka. Implement the outbox pattern for reliable event publishing, saga pattern for distributed transactions, and CQRS for read scaling.

Project: E-commerce backend with Order Service, Inventory Service, Payment Service, and Notification Service — all communicating through Kafka topics. Implement the saga pattern for order fulfillment.
Microservices Outbox Pattern Saga Pattern CQRS
MILESTONE 09
Performance & Production Tuning
Benchmark your cluster. Tune producer and consumer configurations for your target SLOs. Set up monitoring with Prometheus + Grafana. Configure JMX metrics, create consumer lag alerting, and build operational runbooks.

Project: Run kafka-producer-perf-test.sh and kafka-consumer-perf-test.sh. Build a Grafana dashboard tracking throughput, consumer lag, and broker health. Document tuning decisions.
Prometheus Grafana JMX Metrics Benchmarking
MILESTONE 10
Architecture Design & Security
Design event schemas that can evolve. Implement TLS encryption + SASL authentication + ACL authorization. Plan partition strategies for your data. Design for multi-datacenter with MirrorMaker 2. Create disaster recovery procedures.

✅ Expert Level Achieved when: You can design a Kafka-based architecture for a new system, justify partition/replication choices, implement security, operate in production, and debug issues under pressure.
TLS/SASL ACLs MirrorMaker 2 Disaster Recovery Architecture
